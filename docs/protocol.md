# RS485通讯协议
##### 1. 通讯波特率115200（数据位: 8，校验位: 0， 停止位: 1）
##### 2. 小端字节序
##### 3. 默认设备地址：0x01，0xff为广播地址
##### 4. 数据帧格式：
   - 帧头：1字节，主机发送的协议头为0xAE，从机应答的协议头为0xAC；
   - 包序号：1字节，从机应答的包序号与主机发送的包序号相同；
   - 设备地址：1字节, 地址范围为0-0xFF；0x00为广播地址；0xFF为公共地址；
                        广播地址：从机均能收到指令并执行指令，但不应答主机；
                        公共地址：从机均能收到指令并执行指令，但应答主机；RS48总线特性决定，当总线连接多个电机时，不能使用公共地址；
   - 命令码：1字节；
   - 数据长度：1字节；
   - 数据：N字节（根据数据长度而定），0-248字节；
   - 校验码：2字节，协议头-数据字段的CRC16校验，低字节在前，高字节在后。

##### 5. 控制命令
###### 系统:
- 0x00: 重启从机，主控制器发送该命令包后，从机立即重启不应答主控制器
- 0x0A: 读取Boot、软件、硬件、RS485协议和CAN协议版本及UID；
- 0x0B: 读单圈绝对值角度、多圈绝对值角度、速度、Q轴电流等实时信息；
- 0x0F: 清除故障；
###### 参数:
- 0x10: 读取用户参数；
- 0x11: 写入并保存用户参数；
- 0x12: 读取电机硬件参数；
- 0x13: 写入并保存电机硬件参数；
- 0x14: 读取运动控制参数；
- 0x15: 写入运动控制参数但不保存；
- 0x16: 写入并保存运动控制参数；
- 0x1D: 设置当前位置为原点；
- 0x1E: 编码器校准；校准过程中电机需空载，且不能干扰电机转动；
- 0x1F: 参数恢复默认值；
###### 控制：
- 0x20: Q轴电流控制；（力矩 = 力矩常数*Q轴电流）
- 0x21: 速度控制；
- 0x22: 绝对值位置控制；
- 0x23: 相对位置控制；
- 0x24: 电机按照最短的距离回到设定的原点，旋转的角度不大于180度；
- 0x2E: 抱闸控制开关输出控制；
- 0x2F: 关闭电机输出，电机进入自由态不受控制（电机上电后为该状态）；
